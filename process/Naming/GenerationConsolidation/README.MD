# Generation and Consolidation - Processo Naming

> Produz alternativas de nomes e consolida uma long list coerente com o briefing

## Objetivo do processo

Gerar opções de nomes alinhadas ao briefing, registrar racional e categoria de cada alternativa, eliminar candidatos incompatíveis e entregar uma long list organizada para os filtros de viabilidade.

## Quando executar

- Após concluir `process/Naming/ResearchInputs` com insumos aprovados.
- Sempre que for iniciar sessões criativas (individuais, coletivas ou com parceiros).
- Ao atualizar um portfólio de nomes (submarcas, campanhas, programas) com novos requisitos.

## Principais entregáveis

1. `longlist-naming.MD` com alternativas categorizadas, racional e notas fonéticas.
2. Log das sessões criativas (`log-sessoes-criativas.MD`) com técnicas utilizadas e aprendizados.
3. Lista de termos eliminados e motivos (`descartes-naming.MD`).
4. Relatório de priorização (`priorizacao-longlist.MD`) destacando candidatas fortes por critério.

## Dependências e conexões

- **Processo anterior:** `ResearchInputs`.
- **Estratégia associada:** `strategies/Naming` (Etapa Geração e consolidação).
- **Templates base:** `strategies/Naming/_DATA/templates/longlist-naming.MD`, `strategies/Naming/_DATA/templates/matriz-inspiracao.MD` (referência).
