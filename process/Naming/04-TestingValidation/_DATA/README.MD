# _DATA - Testing and Validation (Naming)

Armazene aqui todos os materiais utilizados e gerados durante a etapa de testes e validação.

## Estrutura sugerida

```
_DATA/
  04-TestingValidation/
    roteiro-teste.MD
    registro-participantes.MD
    registro-testes.MD
    relatorio-testes-naming.MD
    recomendacao-nome.MD
    materiais-apresentacao/
    evidencias/
      gravações/
      prints/
      planilhas/
```

## Templates recomendados

- `strategies/Naming/_DATA/templates/roteiro-teste.MD`
- Instrumentos (forms, planilhas, scripts) aprovados pelo time de pesquisa.

## Boas práticas

- Nomeie arquivos com padrão `YYYY-MM-DD_nome.MD`.
- Salve evidências (vídeos, áudios, prints) na pasta `evidencias/` com descrição.
- Registrar consentimento quando aplicável (gravações, pesquisas externas).
