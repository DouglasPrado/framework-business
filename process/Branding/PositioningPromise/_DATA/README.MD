# _DATA - Positioning and Promise

Diretório para armazenar todos os artefatos gerados durante o processo PositioningPromise.

## Organização sugerida

```
_DATA/
  PositioningPromise/
    posicionamento.MD
    mensagens-persona.MD
    matriz-objeções.MD
    validacao-posicionamento.MD
    feedback-log.MD
    agenda-testes.MD
```

## Templates recomendados

- `strategies/Branding/_DATA/templates/plataforma-de-marca.MD` (referência para manter alinhamento).
- `strategies/Branding/_DATA/templates/diagnostico-marca.MD` (consulta ao histórico).
- Novos templates adicionados neste processo (consulte `_DATA/templates/` quando criados).

## Boas práticas

- Siga padrões de nomenclatura `YYYY-MM-DD_nome-arquivo.MD`.
- Versões antigas devem ser movidas para subpasta `archive/` com justificativa.
- Registre fontes de dados (entrevistas, planilhas, dashboards) com links e responsáveis.
