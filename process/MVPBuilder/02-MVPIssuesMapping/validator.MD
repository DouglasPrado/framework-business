# Validador - Mapeamento de Issues para MVP

## Propósito

Este documento fornece critérios objetivos para validar que o processo de Mapeamento de Issues foi executado com qualidade.

## Validação Geral

- [ ] Todas as 25 tarefas foram executadas
- [ ] Tempo total: 3-4 dias (máximo 5)
- [ ] Repositório criado e acessível
- [ ] Projeto Next.js rodando
- [ ] Board kanban configurado

## Validação de Issues

### Quantidade
- [ ] Mínimo 15 issues criadas
- [ ] Issues P0 (setup): 7-8
- [ ] Issues P1 (core): 3-5
- [ ] Issues P2 (secundárias): 4-9

### Qualidade
- [ ] Todas as issues seguem template padrão
- [ ] Critérios de aceite claros em cada issue
- [ ] Todas as issues têm estimativa de tempo
- [ ] Dependências mapeadas
- [ ] Labels aplicadas (tipo, área, prioridade)

### Priorização
- [ ] Issues classificadas em P0, P1, P2
- [ ] Dependências não criam ciclos
- [ ] Ordem de execução é viável

## Validação de Timeline

- [ ] Timeline total calculada
- [ ] Timeline está em 6-8 semanas
- [ ] Se excede, ajustes foram feitos

## Validação de Milestones

- [ ] 4 milestones criados
- [ ] Todas as issues atribuídas a milestones
- [ ] Distribuição equilibrada

## Validação de Board

- [ ] GitHub Projects configurado
- [ ] Todas as issues no board
- [ ] Colunas criadas (Backlog, To Do, In Progress, Review, Done)
- [ ] Automações ativas

## Critérios de Aprovação

O processo está APROVADO quando:
- [ ] Todos os critérios acima atendidos
- [ ] Time técnico validou viabilidade
- [ ] Timeline realista
- [ ] Issues prontas para execução
