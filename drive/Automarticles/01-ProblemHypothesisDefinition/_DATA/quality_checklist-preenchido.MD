## 1. Específica

A hipótese identifica claramente quem, o quê e por quê, sem ser vaga ou genérica?

### Verificações:

- [x] **QUEM é específico**
  - Consigo descrever uma pessoa específica que corresponde ao QUEM? Sim
  - Posso visualizar quem é esse usuário (não é apenas "empresas" ou "usuários")? Sim
  - A descrição inclui contexto relevante (papel, situação, características)? Sim

- [x] **RESULTADO é específico**
  - Consigo observar ou medir quando o resultado é alcançado? Sim
  - O resultado é claro o suficiente para validar? Sim
  - Não é vago como "melhorar" ou "otimizar" sem detalhes? Sim

- [x] **DOR é específica**
  - A dor é concreta, não abstrata? Sim
  - Consigo explicar como a dor se manifesta no dia a dia? Sim
  - Há detalhes sobre o custo (tempo, dinheiro, oportunidade)? Sim

### Perguntas de teste:

1. Consigo descrever uma pessoa real que corresponde ao QUEM?  
   - [x] Sim  [ ] Não

2. Consigo explicar como saberia que o RESULTADO foi alcançado?  
   - [x] Sim  [ ] Não

3. Consigo dar exemplos concretos de quando a DOR ocorre?  
   - [x] Sim  [ ] Não

**Critério ESPECÍFICA:** [x] PASSA  [ ] FALHA

## 2. Valiosa

O RESULTADO é algo que os usuários ativamente querem e valorizariam?

### Verificações:

- [x] **RESULTADO é desejável**
  - Os usuários buscariam ativamente alcançar esse resultado? Sim
  - É importante para eles, não apenas "bom ter"? Sim
  - Faz diferença real no trabalho ou vida deles? Sim

- [x] **Prioridade alta**
  - É uma prioridade suficientemente alta para impulsionar ação? Sim
  - Usuários dedicariam tempo ou recursos para isso? Sim
  - Está entre os top 3-5 problemas/necessidades deles? Sim

- [x] **Disposição para pagar/mudar**
  - Usuários pagariam por uma solução? Sim
  - Mudariam comportamento ou processo atual? Sim
  - Investiriam esforço para adotar uma solução? Sim

### Perguntas de teste:

1. Se eu oferecesse isso de graça, quantos usuários realmente usariam regularmente?  
   - [x] Muitos (50%+)  [ ] Alguns (20-50%)  [ ] Poucos (<20%)

2. Os usuários pagariam por alcançar esse resultado?  
   - [x] Definitivamente  [ ] Provavelmente  [ ] Improvável  [ ] Definitivamente não

3. É uma prioridade alta ou baixa para os usuários?  
   - [x] Alta (Top 3)  [ ] Média (Top 5-10)  [ ] Baixa (Nice to have)

**Critério VALIOSA:** [x] PASSA  [ ] FALHA

## 3. Dolorosa

A DOR é significativa o suficiente para motivar os usuários a buscar soluções?

### Verificações:

- [x] **Severidade**
  - A dor tem consequências significativas? Sim
  - Não é apenas um pequeno inconveniente? Sim
  - O custo da dor é claro (tempo, dinheiro, frustração, risco)? Sim

- [x] **Frequência**
  - A dor ocorre com frequência suficiente para importar? Sim
  - Não é um evento único ou extremamente raro? Sim
  - Usuários experimentam isso regularmente? Sim

- [x] **Reconhecibilidade**
  - Os usuários reconhecem que têm esse problema? Sim
  - Eles admitiriam ter essa dor se perguntados? Sim
  - Usariam linguagem similar ao descrever o problema? Sim

### Perguntas de teste:

1. Qual é a severidade da dor?  
   - [x] Alta (Crítica, bloqueante, custosa)  
   - [ ] Média (Frustrante, algum custo)  
   - [ ] Baixa (Inconveniente ocasional)

2. Com que frequência a dor ocorre?  
   - [x] Diária/Semanal  
   - [ ] Mensal  
   - [ ] Trimestral ou menos

3. Os usuários buscariam ativamente soluções para essa dor?  
   - [x] Sim, definitivamente  
   - [ ] Provavelmente  
   - [ ] Improvável

**Critério DOLOROSA:** [x] PASSA  [ ] FALHA

## 4. Testável

Você pode validar a hipótese através de conversas ou observações com usuários reais?

### Verificações:

- [x] **Usuários acessíveis**
  - Consigo encontrar usuários para conversar? Sim
  - Sei onde encontrá-los (canais, comunidades, eventos)? Sim
  - Posso acessar 10+ usuários em 1-2 semanas? Sim

- [x] **Perguntas formuláveis**
  - Consigo formular perguntas que testariam a hipótese? Sim
  - As perguntas revelariam se a hipótese é verdadeira ou falsa? Sim
  - Não requer pesquisa de mercado extensa antes de testar? Sim

- [x] **Critérios claros**
  - Saberia reconhecer evidência a favor da hipótese? Sim
  - Saberia reconhecer evidência contra a hipótese? Sim
  - Há critérios claros de validação vs invalidação? Sim

### Perguntas de teste:

1. Consigo listar 3+ lugares onde encontrar usuários que correspondem ao QUEM?  
   - [x] Sim  [ ] Não  
   - Lugares: LinkedIn, grupos de Facebook, eventos de marketing digital

2. Consigo formular 5 perguntas para testar essa hipótese?  
   - [x] Sim  [ ] Não

3. Em quanto tempo consigo conversar com 10 usuários?  
   - [x] 1 semana  [ ] 2 semanas  [ ] 1 mês  [ ] Mais de 1 mês

**Critério TESTÁVEL:** [x] PASSA  [ ] FALHA

## 5. Focada

A hipótese aborda um problema claro, não múltiplos problemas não relacionados?

### Verificações:

- [x] **Um problema central**
  - A hipótese foca em um problema claro? Sim
  - Não tenta resolver múltiplos problemas não relacionados? Sim
  - Há um RESULTADO principal, não 3-4 resultados diferentes? Sim

- [x] **Contexto coerente**
  - QUEM, RESULTADO e DOR estão todos relacionados? Sim
  - Fazem sentido juntos no mesmo contexto? Sim
  - Não são elementos desconexos forçados juntos? Sim

- [x] **Concisão**
  - A declaração é concisa e direta? Sim
  - Consegue-se remover palavras sem perder clareza? Sim
  - Cada palavra contribui para especificidade? Sim

### Perguntas de teste:

1. Quantos problemas principais a hipótese tenta resolver?  
   - [x] 1 problema (focada)  
   - [ ] 2-3 problemas (dispersa)  
   - [ ] 4+ problemas (muito dispersa)

2. Todos os elementos (QUEM, RESULTADO, DOR) fazem sentido juntos?  
   - [x] Sim, estão claramente relacionados  
   - [ ] Parcialmente relacionados  
   - [ ] Parecem desconexos

3. A declaração é concisa ou excessivamente longa/complexa?  
   - [x] Concisa (1 frase clara)  
   - [ ] Razoável (1 frase, um pouco longa)  
   - [ ] Complexa (múltiplas orações, difícil de seguir)

**Critério FOCADA:** [x] PASSA  [ ] FALHA
